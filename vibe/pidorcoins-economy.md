## 🎯 Варианты траты койнов

### 1. Защита от пидора (Immunity Shield)

**Описание:** Игрок активирует защиту на 1 день. Если его выбирает рандом, происходит перевыбор.

| Параметр | Значение |
|----------|----------|
| Стоимость | 10 койнов |
| Длительность | 1 день |
| Ограничение | Не чаще 1 раза в неделю |

**Статус:** ✅ **РЕАЛИЗОВАНО** (2026-01-08, улучшено 2026-01-09)

**Реализация:**
- ✅ Создана модель `GamePlayerEffect` с полями `immunity_year`, `immunity_day`, `immunity_last_used` (изолировано по играм через `game_id`)
- ✅ Хранение на year+day вместо datetime для устранения багов с временем суток (v3)
- ✅ Интегрировано в `pidor_cmd` - перевыбор при активной защите
- ✅ Покупка через интерактивное меню `/pidorshop`
- ✅ Начисление койнов защищенному игроку при срабатывании защиты

### 5. Перевыборы (Reroll)

**Описание:** После объявления пидора дня любой участник может заплатить койны, чтобы запустить перевыбор. Текущий "победитель" сбрасывается, и выбирается новый пидор дня.

| Параметр | Значение |
|----------|----------|
| Стоимость | 15 койнов |
| Лимит | 1 перевыбор на чат в день |
| Временное окно | 5 минут после объявления |
| Участие победителя | Да, участвует в перевыборе |

**Статус:** ✅ **РЕАЛИЗОВАНО** (2026-01-22, улучшено 2026-01-23)

- Текущий результат аннулируется, проводится новый розыгрыш
- Кнопка исчезает после использования или через 5 минут
- Если инициатор сам становится пидором — это "карма" 😈
- **ВАЖНО:** Оба победителя (старый и новый) получают награду в койнах!

### 7. Передача койнов (Transfer)

**Описание:** Игрок может передать часть своих койнов другому участнику чата. При передаче взимается комиссия, которая идёт в банк чата.

| Параметр | Значение |
|----------|----------|
| Комиссия | 10% (минимум 1 койн) |
| Куда идёт комиссия | В банк чата |
| Минимум передачи | 2 койна |
| Максимум передачи | Без ограничений |
| Кулдаун | 1 передача в день (по `year+day`, как остальные эффекты) |
| Интерфейс | Через магазин `/pidorshop` |

**Статус:** ✅ **РЕАЛИЗОВАНО** (2026-01-22)

**Механика:**
```
Передача 50 койнов:
- Отправитель теряет: 50 койнов
- Получатель получает: 45 койнов (50 - 10%)
- Банк чата получает: 5 койнов (комиссия)
```

**Особые случаи:**
- Нельзя передавать себе
- Нельзя передавать больше, чем есть на балансе
- Получатель должен быть зарегистрирован в игре
- Кулдаун проверяется по `year+day` (можно передать в 23:50 и в 00:10 следующего дня)

**Реализация:**
- ✅ Создана модель `ChatBank` для хранения баланса банка чата
- ✅ Создана модель `CoinTransfer` для истории переводов и отслеживания кулдауна (с полями `year`, `day`)
- ✅ Создан сервис `transfer_service.py` с функциями:
  - `calculate_commission()` - расчёт комиссии 10% (минимум 1 койн)
  - `has_transferred_today()` - проверка кулдауна по year+day
  - `can_transfer()` - проверка возможности передачи
  - `get_or_create_chat_bank()` - получение/создание банка чата
  - `execute_transfer()` - выполнение передачи с комиссией
- ✅ Интегрировано в магазин `/pidorshop` с пунктами "💸 Передать койны" и "🏦 Банк чата"
- ✅ Созданы callback-обработчики:
  - `handle_shop_transfer_callback()` - показ списка игроков
  - `handle_shop_transfer_select_callback()` - выполнение передачи
  - `handle_shop_bank_callback()` - показ баланса банка
  - `handle_shop_back_callback()` - возврат в меню магазина
- ✅ Передача выполняется на половину баланса отправителя (упрощённая версия)

**Использование банка (будущее):**
- Еженедельный розыгрыш среди активных игроков
- Джекпот для победителя с серией побед
- Награда за достижения
- Специальные события

### 10. Предсказание (Prediction)

**Описание:** Сделать предсказание, кто станет пидором в следующем розыгрыше. Количество кандидатов зависит от размера чата для балансировки вероятности.

| Параметр | Значение |
|----------|----------|
| Стоимость | 3 койна |
| Награда | +30 койнов за правильное предсказание |
| Ограничение | 1 предсказание в день |
| Кандидатов | `ceil(количество_игроков / 10)` |
| Self-prediction | ✅ Разрешено |

**Динамическое количество кандидатов:**

| Игроков в чате | Кандидатов | Вероятность угадать |
|----------------|------------|---------------------|
| 1-10 | 1 | 10-100% |
| 11-20 | 2 | 10-18% |
| 21-30 | 3 | 10-14% |
| 31-40 | 4 | 10-13% |

**Формула:** `количество_кандидатов = ceil(количество_игроков / 10)`

Это выравнивает вероятность угадывания примерно до 10-15% для всех размеров чатов!

**Статус:** ✅ **РЕАЛИЗОВАНО** (2026-01-08, улучшено 2026-01-09, динамические кандидаты 2026-01-22, UX улучшения 2026-01-22)

### Баг: Достижение "Первая кровь" не учитывает год

**Описание проблемы:**
Функция `check_first_blood()` в `bot/handlers/game/achievement_service.py` (строки 219-261) проверяет **общее количество побед за всё время**, а не за текущий год. Это означает, что игрок получит достижение "Первая кровь" только при самой первой победе в чате, но не получит его в следующих годах.

**Текущая логика (строки 242-246):**
```python
stmt = select(GameResult).where(
    GameResult.game_id == game_id,
    GameResult.winner_id == user_id
)
wins_count = len(db_session.exec(stmt).all())

if wins_count == 1:  # Проверяет ОБЩЕЕ количество побед
```

**Проблема:**
Если у игрока уже были победы в прошлых годах, `wins_count` будет больше 1, и достижение не будет выдано в новом году.

**Решение:**
Добавить фильтр по году в запрос (строка 244):
```python
stmt = select(GameResult).where(
    GameResult.game_id == game_id,
    GameResult.winner_id == user_id,
    GameResult.year == year  # <-- Добавить эту строку!
)
```

**Последствия исправления:**
- Сейчас 20 февраля 2026 года
- Если исправить сейчас, большинство игроков не получат достижение за 2026 год (т.к. у них уже есть победы в этом году, но `wins_count` будет > 1)
- Но в 2027 году и далее система будет работать корректно
- Это нормальная ситуация при внедрении новой функциональности

**Влияние:**
- Игроки не получают достижение "Первая кровь" в новых годах
- Противоречит концепции ежегодных достижений
- Особенно заметно для активных игроков, которые играют несколько лет

**Приоритет:** Средний (логика не соответствует концепции ежегодных достижений)
